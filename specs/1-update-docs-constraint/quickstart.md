# Quickstart Guide: Documentation Location Constraint

**Feature**: Update Documentation Location Constraint
**Date**: 2025-01-07

## Overview

This guide provides instructions for ensuring all textbook content is properly located within the Docusaurus `/docs` directory according to the established constraints.

## Prerequisites

- Basic understanding of Docusaurus directory structure
- Git access to the repository
- Node.js and npm for testing changes (optional)

## Content Creation Process

### 1. Creating New Chapters

When creating a new chapter, ensure it's placed under the `/docs` directory:

```bash
# Create a new chapter directory under /docs
mkdir docs/chapter-XX-topic-name

# Create the _category_.json file for the chapter
# This enables proper Docusaurus navigation
cat > docs/chapter-XX-topic-name/_category_.json << EOF
{
  "label": "Chapter XX: Topic Name",
  "position": XX,
  "link": {
    "type": "generated-index",
    "description": "Brief description of the chapter content."
  }
}
EOF
```

### 2. Creating New Lessons

When creating lesson files, ensure they're placed within the appropriate chapter directory under `/docs`:

```bash
# Create lesson file within the chapter directory (under /docs)
cat > docs/chapter-XX-topic-name/lesson-Y-descriptive-title.md << EOF
---
title: Lesson Title
sidebar_position: Y
description: Brief description of the lesson content
---

# Lesson Title

## Learning Objectives

- Objective 1
- Objective 2
- Objective 3

## Content

[Main content with clear headings and explanations]

## Examples

[Practical examples or illustrations]

## Key Takeaways

- Takeaway 1
- Takeaway 2
- Takeaway 3
EOF
```

### 3. Updating Navigation

When adding new content, update `sidebars.js` to reference the new files under `/docs`:

```javascript
// In sidebars.js
module.exports = {
  tutorialSidebar: [
    {
      type: 'autogenerated',
      dirName: '.', // This automatically generates sidebar from /docs structure
    },
  ],
};
```

Or manually reference specific docs paths:

```javascript
// In sidebars.js
module.exports = {
  tutorialSidebar: [
    'intro',
    {
      type: 'category',
      label: 'Chapter Title',
      items: [
        'chapter-XX-topic-name/lesson-1-descriptive-title',
        'chapter-XX-topic-name/lesson-2-descriptive-title',
      ],
    },
  ],
};
```

## Validation Process

### 1. Directory Structure Validation

Verify that all content is properly organized under `/docs`:

```bash
# List all .md and .mdx files under /docs
find docs -name "*.md" -o -name "*.mdx" | sort

# Verify chapter directories exist under /docs
find docs -maxdepth 1 -type d | grep -E "docs/chapter"
```

### 2. Navigation Link Validation

Ensure all navigation references point to content under `/docs`:

```bash
# Check sidebar.js for references to /docs paths
grep -r "docs/" sidebars.js
```

### 3. Content Location Verification

Verify no content exists outside the `/docs` directory:

```bash
# Find any .md or .mdx files outside of docs/
find . -path "./docs" -prune -o -name "*.md" -print
find . -path "./docs" -prune -o -name "*.mdx" -print
```

## Migration Process

### 1. Identify Content Outside /docs

```bash
# Find content files outside the docs directory
find . -path "./docs" -prune -o -name "*.md" -print
find . -path "./docs" -prune -o -name "*.mdx" -print
```

### 2. Move Content to /docs

For content found outside `/docs`, move it to the appropriate location within `/docs`:

```bash
# Example: Move a file to a chapter subdirectory under /docs
mv path/to/external/content.md docs/chapter-XX-topic-name/content.md

# Update any navigation references to point to the new location
# This may require updating sidebars.js or other navigation files
```

## Best Practices

- Always create content under the `/docs` directory
- Organize content in logical chapter subdirectories
- Update navigation references to match the `/docs` structure
- Validate content locations regularly
- Use consistent naming conventions for files and directories
- Maintain proper frontmatter in all lesson files
